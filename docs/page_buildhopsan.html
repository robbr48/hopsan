How to Build Hopsan and Dependencies

Hopsan depends on a few third party libraries. Some of these are required others are optional. When you build Hopsan through QtCreator the Hopsan project files will look for the dependencies in the Dependencies directory. For your convenience there are scripts that will automatically unpack, configure and build each dependency, provided that the build requirements are installed.

These scripts can be found in the Dependencies directory. See Hopsan Dependencies and Licenses for information on which dependencies are required and which are optional.

Note! Some dependencies do not require compilation. It is enough that the needed files are present in the Dependencies directory.
Note! Some scripts are missing on POSIX platforms, is those cases you should use your systems version of the libraries instead. Install them from you distribution package database.

If you wish to see the old (outdated) instructions for manual dependency building they are kept here Building the Hopsan Dependencies manually

Build Hopsan in QtCreator (all OS)

  1. Build the required and desired optional dependencies first
  2. Open the HopsanNG.pro project file. Make sure that you are using the same qmake version and build tool chain as you used to build the dependencies.
  3. Go to Projects->Build and under Build Steps enter the Make argument: -j4 to speed up compilation using 4 cores. (Optional)
  4. It is recommended to activate shadow build for both debug and release, (you will likely be able to use the default shadow build location) Shadow build means that all of the build files will be put someplace else to avoid clutter
  5. qmake will automatically search for the previously build dependencies in the Dependencies directory. Each dependency have a .pri file that specify the needed compilation and linking variables.
  6. Before you compile, run qmake on the HopsanNG project (right click), to see the qmake messages, (you will see if all dependencies were found correctly). This information is shown in the compile tab.
  7. Build the project in either debug or release mode
  8. If you want to start HopsanGUI through Qt Creator, go into the left panel Projects->Run and change the Run configuration.
